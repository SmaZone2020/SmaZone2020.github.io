## 一个根本性的问题

> 如果没有任何预设的社会规则，没有"合作"的指令，没有"语言"的定义——
> 仅仅给定生存压力、有限资源、局部感知和行动能力——
> 复杂行为能否从零开始涌现？

这就是 ZHI 试图回答的问题。

**GitHub：** [Yuxi-IT/ZHI](https://github.com/Yuxi-IT/ZHI)

---

## 为什么叫"零假设"

ZHI 的全称是 **Zero-Hypothesis Intelligence Ecosystem**（零假设智能生态系统）。

"零假设"的意思是：我们尽可能不在系统中埋入任何关于"智能应该是什么样"的假设。没有友方/敌方，没有阵营，没有社会关系，没有语言语义，没有合作规则。

我们只定义七件事：**生存、资源、空间、环境、感知、需求、行动**。仅此而已。其余一切——如果出现了的话——都是涌现的。

---

## 世界是什么样的

想象一个 **64×64 的网格世界**，上面生活着最多 64 个 Agent，以及不断生长和腐烂的植物。

**Agent 拥有：**

- 340 维观察向量（7×7 方向性视野 × 6 通道 + 46 维自身状态）
- CNN + GRU 混合神经网络大脑（PPO 在线训练）
- 7 个可遗传身体参数（Size/Speed/Strength/VisionRange/Fat/ColdResist/HeatResist）
- 统一的 Energy 货币——所有动作都扣能量，只有进食能获取
- 8 个离散动作：上下左右、吃、攻击、释放化学信息素、饮水

**世界提供：**

- 24 小时昼夜循环（12 现实分钟 = 1 天），温度正弦变化 5°C~35°C
- 动态水循环：降雨 → 地表水流 → 蒸发 → 地下水交换
- 风与气压系统：温度差 → 气压差 → 风 → 气味输送 + 温度平流
- 8 种生态区（水域/河岸/荒漠/草原/丛林/湿地/高地/谷地）
- 四阶段植物生命周期（Seed → Sprout → Adult → Decay）
- 自然灾害：洪水、干旱、热浪、寒潮
- 旱/雨季节律（~10000 tick 周期）

## 核心设计：能量经济

ZHI 最核心的设计原则是 **能量经济**——一切行为都有代价，一切生存策略都必须通过能量核算。

```
移动 → 扣 Energy（平地 0.5，浅水 +1.0，深水 +2.5）
攻击 → 扣 8 Energy / BodyStrength
释放信号 → 扣 3 Energy
饮水 → 恢复 40 Water
进食 → 每 tick 从植物/尸体提取 Energy
基础代谢 → 每 tick 扣 Energy（受 Size/温度/年龄/污染影响）
```

冷 → 代谢加速 → 更多能量消耗 → 死亡。
热 → 水分流失 → 脱水 → 能量惩罚 → 死亡。
Energy ≤ 0 → 死亡。

Agent 必须学会：什么时候移动、什么时候进食、什么时候攻击、什么时候饮水、什么时候释放信号。每一步都是经济决策。

## 遗传与进化

Agent 不会永生。5000 tick 后开始衰老，8000 tick 强制死亡。

但死亡不是终点。Agent 死亡后 5 秒自动重生——保留网络权重（策略智慧），清零短期记忆（GRU 隐藏状态）。

Agent 可以通过无性繁殖产生后代。生殖采用**妊娠机制**：亲代进入孕期（持续 N tick，移速减半，受击伤害 ×1.4），孕期结束产下子代。子代继承亲代的 Genome，经过高斯变异（std=0.05）后获得略有不同的身体参数。

这就构成了达尔文式的进化循环：

```
变异 → 差异化的生存策略 → 自然选择（能效高的活更久） → 繁殖 → 更多变异
```

## 生态位从底层涌现

ZHI 有一个生态位观察系统。每 200 tick，系统根据每个 Agent 的饮食历史自动分类：

| 生态位               | 判定条件       | 标签颜色 |
| ----------------- | ---------- | ---- |
| 🌿 植食 (Herbivore) | 食物进食 > 70% | 绿色   |
| 🥩 肉食 (Carnivore) | 攻击次数 > 50% | 红色   |
| 🦴 腐食 (Scavenger) | 尸体进食 > 50% | 黄色   |
| 🍽 杂食 (Omnivore)  | 均衡         | 灰色   |

关键点：**Agent 不会被预设角色**。一个 Agent 成为"肉食者"不是因为程序员写了 `if (role == "predator")`，而是因为它发现攻击其他 Agent 能带来更高的能量回报。

## 技术架构

```
┌─────────────────────┐
│  ZHI.Dashboard      │  React 19 + TypeScript + Vite
│  Web UI / 3D 视图   │  Three.js (react-three-fiber)
│  实时 WebSocket 推送 │  HeroUI + TailwindCSS
└────────┬────────────┘
         │ WebSocket
┌────────┴────────────┐
│  ZHI.Watcher        │  .NET 10 + TorchSharp
│  世界引擎 (CPU)      │  CosmosEngine: 15+ 分部类
│  PPO 训练 (GPU)      │  CNN+GRU 混合网络 BatchBrain
│  SQLite 持久化       │  谱系追踪 / 世代保存
└────────┬────────────┘
         │
┌────────┴────────────┐
│  ZHI.Core            │  GRUBrain / RND / MAP-Elites
│  神经网络核心         │  PPOBuffer (GPU 显存预分配)
└──────────────────────┘
```

**为什么选择 .NET + TorchSharp？**

GPU 推理和训练直接在 C# 进程中完成，没有 Python 桥接开销。PPO 经验缓冲预分配在 GPU 显存上，每步增量写入，避免了 PCIe 批量传输。所有 64 个 Agent 共享一个 BatchBrain，一次前向传播完成全部推理。

## 3D 可视化

Dashboard 提供 2D Canvas 和 Three.js 3D 两种视图，一键切换：

- **地形**：高度位移渲染（HeightMap 0-255 → 5 单位高差）
- **生态区**：8 色顶点着色
- **Agent/植物/尸体**：InstancedMesh 实例化渲染（单次 draw call）
- **水面**：半透明蓝色层
- **光照**：跟随昼夜循环（方位角 + 色温 + 强度变化）
- **叠加层**：气味、化学场、温度热力图、地表水、地下水、养分、气压、风

## 我们观察到什么了

项目还在早期阶段，但已经有一些有趣的信号：

- Agent 确实学会了在 Energy 低时优先觅食而非攻击
- 高温天气下 Agent 会自发靠近河流
- 化学信息素的使用率在代际间逐步上升
- 某些 Agent 发展出了明显的"定居"策略（长期在同一区域活动）

但这些是否构成真正的"涌现"，还需要更长时间的观察。

## 关于项目

ZHI 受 [Artificial Life](https://en.wikipedia.org/wiki/Artificial_life) 领域启发，试图用最简单的规则观察最复杂的现象。

项目的核心假设是：

> 如果一个行为在最小系统中都无法涌现，那它在更大更复杂的系统中也不太可能是"真的"涌现——更可能是人的偏见在起作用。

如果这个假设成立，ZHI 将展示一条新路径：不是通过更大的模型、更多的数据、更精巧的对齐技术，而是通过构建足够丰富的环境，让智能自己长出来。

如果这个假设不成立，ZHI 同样有价值——它会告诉我们涌现的边界在哪里。

**无论哪种结果，都是答案。**
